# 轻量Imgur代理图床
基于 Cloudflare Worker 构建的无服务器公共图床，前端纯静态页面，零后端、零服务器成本，
支持图片拖拽上传、批量分页加载图库、复制直链、复制Markdown、下载原图，点击缩略图新标签打开高清原图。

## ✨ 项目特性
- 完全免费：依托 Cloudflare 免费额度，无需域名备案、无服务器开销
- 双Worker架构：上传Worker + 资源代理Worker，解决Imgur国内访问屏蔽问题
- 极简UI：全局直角设计、固定顶部导航栏、瀑布流多列自适应布局
- 上传交互：点击/拖拽上传，实时进度条，上传完成直接展示纯代理直链，一键复制
- 图库功能：滚动无限分页加载、下拉加载更多媒体文件
- 快捷操作：每张卡片自带下拉菜单，支持下载原图、复制Markdown图片链接
- 预览方案：点击图片直接新开浏览器标签查看无损高清原图，无冗余弹窗
- 移动端完美适配：手机自动切换2列瀑布流，上传浮层自适应屏幕尺寸
- 轻量化无冗余：已移除多余全屏预览弹窗，代码干净，加载速度快

## 📦 部署前置
需要部署两个 Cloudflare Worker 脚本：
1. Imgur 上传 Worker：负责接收文件并上传至Imgur存储
2. 资源代理 Worker：代理Imgur原图/缩略图，解决访问污染问题

## 🚀 部署教程
### 1. 部署后端双Worker脚本
后端上传、代理 Worker 完整部署图文教程参考：
https://waiyiyuyan.github.io/2026/06/15/imgur-bed/

1. 打开上方教程链接，按照步骤依次部署「上传 Worker」与「资源代理 Worker」
2. 部署完成后记录两个 Worker 的访问地址（自定义域名/默认workers.dev域名均可）

### 2. 修改前端配置
打开 `main.js` 修改顶部配置项，替换为你自己的Worker域名：
```js
const UPLOAD_WORKER = "你的上传Worker域名";
const PROXY_WORKER = "你的代理Worker域名";
```

### 3. 静态页面部署（3种任选）
1. Cloudflare Pages：上传 index.html + style.css + main.js 一键部署
2. Vercel / Netlify：直接推送仓库自动构建
3. 本地静态服务器：直接打开index.html本地运行（需保证Worker跨域允许）

## 📖 页面使用说明
### 上传图片
1. 点击顶部导航栏「上传图片」打开上传浮窗
2. 两种上传方式：点击区域选择本地文件 / 直接拖拽图片进浮窗
3. 上传过程实时显示进度百分比
4. 上传完成自动展示代理直链，点击「复制链接」一键复制图片地址
5. 上传成功后图片自动插入图库顶部，无需刷新页面

### 图库操作
1. 瀑布流展示所有已上传图片/视频，滚动到底自动加载下一页
2. 点击图片缩略图：浏览器新标签打开高清无损原图
3. 点击卡片右下角 `...` 呼出操作菜单：
   - 下载文件：下载原图到本地
   - 复制 Markdown：复制可直接用于博客的图片语法链接

## ⚙️ 自定义修改配置
### 样式自定义（style.css）
1. 全局圆角：修改 `:root` 内 `--radius`，0=直角，4px/8px为圆角
2. 瀑布流列数：媒体查询 `@media` 中调整 column-count 数值
3. 页面间距、颜色、按钮配色、上传浮窗大小均可直接修改CSS变量

### 功能自定义（main.js）
1. 修改Worker接口地址
2. 调整分页每次加载数量、滚动触底距离
3. 修改复制按钮文字、上传提示文案
4. 可自由开启/关闭弹窗预览、调整交互逻辑
